## Exploiting the __type Meta-field :

- Some implementations block introspection by detecting `__schema` , so we can use other meta-field like `__type` which returns all types in the system.
    
    ```graphql
    query {
      __schema {
        types {
          name
        }
      }
    }
    ```
    
- To enumerate type names :
    
    ```graphql
    {
      __type(name: "UserObject") {
        name
        fields {
          name
          type {
            name
            kind
          }
        }
      }
    }
    ```
    
    - If `User` exists, this reveals all its fields.

# Using Field Stuffing

- **Field Stuffing** is when an attacker sends a **query with many guessed field names** (like brute-force) in the hope that:
    - Some of them exist and **return real data**
    - Or trigger **"Did you mean..."** suggestions (which can leak real field names)
- GraphQL schemas often **mirror the database schema** — so if a `user` table has columns like `email`, `password_hash`, `api_key`, etc., there's a good chance the `User` GraphQL type also has those fields.
- GraphQL field names often follow `camelCase`, even if the backend (DB) uses `snake_case`.
- GraphQL are usually written in *UpperCamelCase*.

```graphql
{
  u: __type(name: "User") { name }
  ut: __type(name: "UserType") { name }
  uo: __type(name: "UserObject") { name }
  p: __type(name: "Profile") { name }
  a: __type(name: "Account") { name }
  o: __type(name: "Order") { name }
  i: __type(name: "Item") { name }
}
```

# Using Field Suggestions

- If you make a typo, GraphQL may inform you with this :
    
    ```graphql
    {
      users{
        nam
      }
    }
    ```
    
    ```json
    {
    	"errors": [
    		{
    			"message": "Cannot query field \"nam\" on type \"UserObject\".
    			Did you mean \"name\"?",
    			"locations": [
    				{
    					"line": 15,
    					"column": 5
    				}
    			]
    		}
    	]
    }
    ```
    

## Understanding the Edit-Distance Algorithm :

- Edit-Distance Algorithm is counting the minimum number of operations (add, remove, replace) needed to convert one string into another.
- For example :
    
    
    | String | Typo | Operations | Edit distance |
    | --- | --- | --- | --- |
    | title | titl | Add e | 1 |
    | content | rntent | Replace r with c, add o | 2 |
- Using the algorithm depend threshold which calculated using `const threshold = Math.floor(input.length * 0.4) + 1;`

## **Optimizing Field Suggestion Use :**

- GraphQL may return multiple fields, so if you entered `ownr` , it will respond with `owner` and `ownerId` .
- No limit to the number of typos a client can send in a single request.

# Abusing Error Messages

## Enabling Debugging :

- When **debug mode** is enabled (usually for development), the GraphQL server includes **extra details** in error responses.
- To enable it :
    
    ```
    http://example.com/graphql?debug=1
    http://example.com/graphql?XDEBUG_SESSION_START=PHPSTORM
    ```
